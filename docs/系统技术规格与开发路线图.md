## 1. 系统全貌概述

### 1.1 一句话核心闭环

**通过 Modbus/TCP 实时采集中央空调设备运行数据 → 时序存储 → 智能分析（能效/故障/联动） → 控制指令下发（启停编排 + 策略联动） → 大屏可视化 + 告警通知，形成 "感知—分析—决策—执行" 的完整能源管理闭环。**

### 2.2 硬件协议对接

| 协议 | 实现方式 | 状态 |
|------|---------|------|
| **Modbus RTU over TCP** (G770 DTU 透传) | `TransparentTransport`：TCP 长连接，自动 drain 残留数据，CRC16 校验 | ✅ 已完成 |
| **自定义协议 (0x68/0x16)** | `CustomTransport`：MAC 地址注册 + LoRa 节点地址 + Modbus 载荷 + 校验和，0xE5FF 注册帧 / 0xE4A1 数据帧 | ✅ 已完成 |
| **Modbus 功能码** | 支持 01/02/03/04（读）、05/06/0F/10（写）、异常码 0x80+ | ✅ 已完成 |
| **字节序支持** | 高位在前 / 低位在前 / 低字在前 | ✅ 已完成 |
| **数据类型解码** | float32(IEEE754)、uint16、int16、int32、uint32、位掩码 | ✅ 已完成 |

## 3. 架构与逻辑审查 (Code Review)

### 3.1 架构评价

**优点：**

| 维度 | 评价 |
|------|------|
| **分层清晰** | Handler → Service → Repository → DB，职责分离良好，接口边界明确 |
| **Gateway 抽象** | `Transport` 接口统一了自定义协议和 DTU 透传的差异，扩展新协议成本低 |
| **安全机制** | 三层防护：限流器 → JWT 认证 → RBAC 权限（40+ 权限码），最近一轮已修复密码重置和 JWT 强制配置 |
| **数据一致性** | 启停步骤使用事务包裹 DELETE+INSERT；告警使用 PostgreSQL 部分唯一索引去重 |
| **容错设计** | 天气服务有 60min 缓存 + 过期回退；网关断线自动重连；goroutine panic 恢复 |
| **可观测性** | 所有关键路径有 `slog` 日志（Info/Warn/Error 分级），控制记录有完整审计日志 |

**缺点与风险：**

| 问题 | 严重度 | 详情 |
|------|--------|------|
| **Handler-Service 边界模糊** | 中 | `dashboard.go` 和 `startup.go` 直接在 handler 中写复杂 SQL（15行 LATERAL JOIN），违反分层原则 |
| **数据库直连 handler** | 中 | `DashboardHandler`、`LogHandler`、`StartupHandler`、`AlarmHandler` 均持有 `*pgxpool.Pool` 直接执行 SQL，跳过了 Repository 层 |
| **缓存机制缺失** | 中 | 大屏数据（`/screen/data`）每次请求执行 8+ SQL 查询，无服务端缓存，并发压力下可能成为瓶颈 |
| **轮询模式** | 中 | 前端当前使用 30s 定时轮询获取实时数据，缺少 WebSocket 推送，实时性与带宽效率不佳 |
| **测试覆盖不足** | 中 | 仅 5 个测试文件（44 测试用例），核心路径（gateway 协议解析、telemetry 数据解码、告警引擎）基本无测试 |
| **单点故障** | 中 | 网关连接由单进程管理，无掉线持久化和自动恢复后的数据补采机制 |
| **Repository 层重复代码** | 低 | Building/Device/Property/Register 四个 Repo 的 CRUD 方法结构几乎相同，未使用泛型提取 |

### 3.2 性能瓶颈预判

| 场景 | 瓶颈点 | 当前策略 | 风险 |
|------|--------|---------|------|
| **大屏并发** | `/screen/data` 每请求 8+ SQL | 无缓存，每次全量查询 | 10+ 并发时 DB 连接池压力大 |
| **遥测大批量** | `device_telemetry` 表写入频率 = 寄存器数 × 设备数 / 3s | pgx Batch Insert | 千设备级可支撑，万级需 partition |
| **告警风暴** | 批量设备同时触发规则 | 部分唯一索引去重 | 可有效防止重复，但写入仍有开销 |
| **网关连接数** | `pollLoop` 中每个 poll 周期遍历所有设备 | 5 并发信号量控制 | 单网关 20+ 设备时，poll 周期可能 > 3s |

### B. 关键技术选型

| 层级 | 技术 | 说明 |

| **语言** | Go 1.22+ | 编译型、高并发、适合网关协议处理 |

| **权限** | RBAC (40+ 权限码) | 角色-权限多对多，支持自定义角色 |
| **前端框架** | React 18 + Vite | TypeScript, TailwindCSS, Recharts, Axios |
| **小程序** | UniApp (Vue 3) | uView UI, Pinia, 单一 API client |
| **天气数据** | wttr.in (免费) | 10s 超时，60min 缓存，支持中英文翻译 |
| **日志** | Go `log/slog` | 结构化日志，Info/Warn/Error 三级 |
